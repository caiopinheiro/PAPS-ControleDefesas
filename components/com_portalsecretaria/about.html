FEATURES
Integration

Integrates fully into the Mambo 4.5 architecture, users must be:

  • Registered to post & view tickets
  • Administrators or Super Administrators to open and process tickets

Attention: config.php file in the component folder must have write (CHMOD777) access

 
Categories

Administrators can define an unlimited number of categories for tickets, with the option to send every category to a different helpdesk administrator. No backend login is required for this.

 
Content Links

For each Ticket or reply to a ticket, an administrator can link a "see also" article (any content) by simply providing the content ID to the ticket or reply. This allows easy streamlining of questions from users that don't take time to read the FAQ or support documents already provided on your Mambo site.

 
Notification

Users and the administrator get notified on posting of new tickets, replies and status changes (new, read, in progress, resolved, closed)

 
Viewing

Users can view all tickets (including closed ones), or open tickets that are in progress. Selecting a ticket displays details, all replies, and the option to change status, priority level, or add a reply.

 
Ticket Status

Users can change status of Tickets to:

  • New
  • Resolved (handy when they found the answer themselves)
  • Closed

Administrators have the extra options to change status to:

  • Read
  • In Progress

 
Attachments

Users and administrators can attach files to tickets and replies. Administrators can configure:

  • Allowed file extensions
  • Maximum size
  • Upload folder and URL Path to attachments

By default, the upload folder is a subfolder called "files" in the component's folder. The CHMOD rights for this folder must be 777.

 
Priorities

Users can change priority of Tickets to:

  • Low
  • Medium
  • High

Administrators have the extra options to change priority to:

  • Top Priority
  • Delete Tickets and replies
